# Budget

Personal finance app — envelope budgeting with item-level tracking, debt management, and net-worth visibility.

## Stack

- **Backend**: Node + TypeScript + Hono + Drizzle + better-sqlite3
- **Frontend**: React + Vite + TanStack Query
- **Database**: SQLite (single file in `./data/app.db`)
- **Validation**: Zod (shared between client and server)

## First-time setup

```bash
# Install dependencies
npm install

# Create the SQLite database from the schema
npm run db:push

# Start dev servers (backend on :3000, frontend on :5173)
npm run dev
```

Open `http://localhost:5173` — you should see "The Ledger" with an empty state.

## Project structure

```
src/
├── server/             # Hono routes, business logic
│   ├── index.ts        # Entry point
│   └── routes/         # One file per resource
├── client/             # React app
│   ├── App.tsx
│   ├── main.tsx
│   ├── components/
│   └── pages/
├── shared/             # Code used by both client and server
│   └── schemas.ts      # Zod schemas for runtime validation
└── db/
    ├── schema.ts       # Drizzle schema (source of truth)
    ├── client.ts       # DB connection
    └── migrations/     # Generated by drizzle-kit
```

## Useful commands

| Command | What it does |
|---|---|
| `npm run dev` | Run server + client with hot reload |
| `npm run db:push` | Sync schema changes to SQLite |
| `npm run db:studio` | Open Drizzle Studio (DB browser) at `local.drizzle.studio` |
| `npm run typecheck` | Check types without building |
| `npm run build` | Production build |
| `npm start` | Run production server |

## Adding your first account

With the dev server running, in another terminal:

```bash
curl -X POST http://localhost:3000/api/accounts \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "AMHFCU Checking",
    "type": "asset",
    "subtype": "checking",
    "startingBalanceCents": 247331
  }'
```

Refresh the browser — your account appears.
